Chapter 3

When introducing a new technology or design, why did Frederick Brooks advise plan to throw one away?: Because you most likely will not get it right the first time.

What is a core dump? Why the use of the word core?: The computer drops everything, completely stops, and then spits out a file reporting the exact contents of its memory at the moment of failure. The word core refers to the computers active memory.

Rather than writing program statements in binary code, 110101110 1001101111, programmers developed a shorthand language called what?: Assembly Language

Adding layers of abstraction, new programming languages were created: Lisp, Cobol, Algol, Basic. Fortran was the first widely used. What kind of program converted Fortran to binary?: It was called a Complier.

Chapter 4

What do front ends and back ends mean to software developers?: The front end is what the user is seeing. The back end is where the result of front end events and inputs go so the computer can make sense of them, save them, and retrieve them.

What did the Lego Hypothesis refer to?: Programs being built of reusable software parts. These software parts would be available everywhere, so that you could take some pre-made pieces of code and plug them in to make some software without doing any coding.

Give one reason why the Lego Hypothesis seems to not work so well.: Unlike Legos, the software objects were not alike enough to be put together willy nilly. They varied enormously in size, function, and in their number of connections to other components. In actual programs, no components are constant.

Chapter 5

What is the three-way trade-off that many software projects struggle to overcome.: There are three major focuses of a project, but you can only fully do two of the three. If you try to do all three, you will either fail or they will all lack.

What is the more recent definition of geek?: A person who has chosen concentration rather than conformity, one who pursues skill and imagination, not mainstream social acceptance.

What does refactoring mean to programmers?: Rewriting a chunk of code to make it briefer, clearer, and easier to read without changing the function.

What is yak-shaving?: A seemingly pointless activity which is actually necessary to solve a problem which solves a problem which, several levels of recursion later, solves the actual problem you were originally working on.

Chapter 6

What is term edge cases referring to in software development?: Extreme possibilities and unlikely scenarios. Often where bugs hide.

Summarize briefly Linus Torvalds advice about large projects given in 2004: You should always start small and think about the details. Never expect to undertake a large project. If you do, you will overdesign and think it is more important than it is. Also, the project should solve a fairly immediate need, and if it doesn't then it is overdesigned.

What does the author state is the ...single most challenging demand of software development?: Communicating abstractions unambiguously; from programmer to machine, from programmer to programmer, and from program to user.

Chapter 9

Structured programming evolved to address what programming practice?: The Go-To Statement. The central idea was to compose a program as a collection of subunits, each of which having only one point of entry and one point of exit.

Was structured programming a solution to the problem of software development?: It was in a way. But it was not the final solution, rather, it laid a solid foundation for more ambitious software to come.

Have any techniques shown to improve the software development process?: Some worked for some projects, but there weren't any that were surefire methods / techniques to improve every software development process.

The waterfall model of programming was/is popular. What were some problems with this model?: It led to delay, confusion and disaster. Everything took forever and nothing worked correctly. Putting large chunks of code together all at the end caused the system to collapse as well.

What are the four tenets of Agile Software Development?: Individuals and interactions over processes and tools. Working software over comprehensive documentation. Customer collaboration over contract negotiation. Responding to change over following a plan.

What did a 2004 study find about the development practices of some two hundred software team leaders?: The most dominant development practice was no practice at all.

What is the Joel Test and what did he say about Microsoft and the Joel Test?: It is a short set of questions that are designed to judge whether a development organization has its act together. Microsoft scores perfectly on the test.

What is Rosenberg’s Law?: Software is easy to make except when you want it to do something new. And the only software that is worth making is software that does something new. Therefore, software is never easy to make.
