What roles in the Chandler project did Michael Toy, John Anderson, Ted Burgess, Mitchell Kapor, and Lou Montulli hold.: They were to find the bugs (open problems / bugs)

What is “Bugzilla?”: A program that tracked all the tasks that needed to be finished

What is OSAF?: Open Source Applications Foundation

What is the projects name?: Chandler

What will the software do?: It's a Personal Information Manager

What is Toy’s keyword for “black hole” bugs?: Scary

What scared Toy so much about Bug 44?: Not knowing how long it would take to fix.

What did Toy refer to as a “snake?”: An important problem that they didn't know how to attack.

In the software world, what does “slippage” mean?: Being late.

Fredrick Brooks was a programming manager for what software project?: IBM's operating system, System/360.

What is Brooks's Law?: Adding manpower to a late software project makes it later.

Brooks found what % of project time was spent writing code?: About 16%.

Brooks found what % of project time was for testing and fixing bugs?: About 50%.

Brooks observed that the unit of effort named “man-month” only applied under what conditions?: When a task can be partitioned among many workers with no communication among them.

What is the difference between source code and the program you install (.exe) on your computer?: Source Code is written and coded into binary files that are used by the computer to display programs.

What is the one “article of faith” that all “open source” or “free” software advocates share?: Software that anyone can tinker with is bound to improve over time in ways that closed software cannot.

What is the difference between a “good” programmer and a “great” programmer?: Good programmers know what to write, and Great programmers know what to rewrite.

Eric Raymond’s book “The Cathedral and the Bazaar” made a distinction between two important project development ideas, briefly contrast them.: The Cathedral approach was carefully crafted by a small amount of highly skilled indivuals with no beta to be released, and the Bazaar approach was an open source code that was available to many programmers that allowed lots of different ideas and approaches to problems.

Has “open source” software project development refuted Brooks’s “mythical man-month” concerns?: No.

What was Andy Hertzfeld’s input when the Chandler project appeared to have stalled?: They lacked "hustle".
